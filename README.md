# 交我学自动填答案 – 浏览器扩展

一个轻量级的 Chrome/Edge 扩展，专为 **交我学**（`jwx.sjtus.cn`）试卷页面的填空题设计，能够一键解析并填入答案。

---

## 📦 安装

### 1. 下载源码
将以下两个文件放入同一个文件夹（例如 `jiawoxue-fill`）：
- `manifest.json`
- `content.js`

### 2. 加载扩展
- **Chrome / Edge / Brave 等 Chromium 内核浏览器**  
  1. 在地址栏输入 `chrome://extensions/`（Edge 用 `edge://extensions/`），回车。  
  2. 打开右上角的 **“开发者模式”** 开关。  
  3. 点击 **“加载已解压的扩展程序”**，选择刚才的文件夹。  
  4. 加载成功后，扩展会出现在列表中。

> ⚠️ 其他浏览器（Firefox、Safari 等）不直接兼容 Manifest V3，需修改配置，本说明不涵盖。

---

## ✨ 功能

- 🧠 **智能解析答案**  
  支持常见答案格式，如：`1) 答案`、`2. 答案`、`3、答案` 等，自动按序号排序并填入。

- 📋 **一键填入**  
  点击页面右下角的「自动填答案」按钮，自动读取剪贴板中的答案文本（若无法读取则弹出输入框供手动粘贴）。

- 🎯 **仅作用于试卷页面**  
  匹配 `https://jwx.sjtus.cn/app/qb/spd*`，不会影响其他网站。

- 🔌 极简权限，仅需 `clipboardRead`（用于读取剪贴板中的答案）。

---

## 🖱️ 使用方法

1. **准备答案文本**  
   将答案按题目顺序整理好，格式见下方【答案格式】一节，复制到剪贴板。

2. **进入试卷页面**  
   打开交我学试卷（网址包含 `/app/qb/spd`），等待所有题目加载完毕。

3. **点击按钮**  
   页面右下角会出现一个蓝色的「自动填答案」按钮，点击它。

4. **完成**  
   扩展会自动读取剪贴板内容并填入所有填空题中，完成后弹出提示。

---

## 🧾 答案格式

答案文本需包含序号，每行一个，支持以下分隔符：

- 序号与答案之间可用 `)`、`.`、`、` 或中文括号分隔。
- 空行、无关说明文字（如“根据视频内容……正确答案如下”）会被自动忽略。
- **答案顺序必须与试卷中填空题的顺序一致**，扩展按序号从小到大填入。

---

## ⚠️ 局限性（已知不足）

- 🚫 **仅支持填空题**  
  当前版本只处理 `<span class="tk">` 下的 `<input>` 元素（即填空题），**不支持**：
  - 选择题（单选/多选）
  - 判断题
  - 拖拽排序等其他交互题型

  > 实现选择题其实更简单（只需 `.click()` 或设置 `checked`），但作者暂时懒得写。  
  > 如果你想扩展功能，欢迎提交 PR 或自行修改 `content.js`，核心逻辑非常容易添加。

- 🔄 依赖页面 DOM 结构  
  如果交我学平台更新了试题的 HTML 结构（例如不再使用 `span.tk input`），扩展可能失效。

- 📋 剪贴板读取可能受限制  
  - 需要浏览器授予 `clipboardRead` 权限（首次点击按钮时会提示）。  
  - 在部分严格的 HTTPS 环境下或用户拒绝权限时，会降级为手动粘贴框。

## 🛠 自定义配置

直接编辑 `content.js` 顶部的变量即可：

- **修改匹配题型**：更改 `querySelectorAll('span.tk input')` 的选择器，可适配其他题型。  
- **调整按钮样式**：修改 `btn.style.cssText` 里的 CSS 属性。  
- **增加题型支持**：在 `fillAnswers` 函数中补充对 `radio`、`checkbox` 的处理逻辑。

修改后，在扩展管理页面点击「刷新」图标重新加载即可生效。

## ❓ 常见问题

**Q: 点击按钮没有反应？**  
A: 确保试卷已完全加载（所有填空输入框出现）。若一直未出现，尝试刷新页面。

**Q: 提示“未能识别出有效答案”？**  
A: 检查答案文本格式是否包含正确的序号和分隔符，并且序号从 1 开始连续。

**Q: 答案数量与填空数量不一致？**  
A: 扩展会弹出确认框，你可以选择继续强制填入（多出的答案会被忽略）或取消操作，检查答案文本是否缺失或多余。

**Q: 为什么不做成自动填写选择题？**  
A: 技术上非常简单，只是因为作者懒。如果你有需要，可以自行在代码中加几行就好了（例如找到选择题 `.question-option input`，按答案索引 `click()`）。欢迎动手改进。
